Introduction to Security concepts and standards for Information Technology


CIA Triad

The CIA triad (Information Security Triad) is a common model that forms the basis for the development of security systems that are used for finding vulnerabilities and methods for creating solutions.

The CIA triad represents the three pillars of information security named Confidentiality, Integrity, and Availability:
- Confidentiality - preserving authorized restrictions on information access and disclosure, including means for protecting personal privacy and proprietary information
- Integrity - guarding against improper information modification or destruction and ensuring information non-repudiation and authenticity
- Availability - ensuring timely and reliable access to and use of information

The security posture of an organization will ideally benefit by the meeting of the three standards.

NIST - MITRE - NCF

The National Institute of Standards and Technology (NIST) was founded in 1901 and is now part of the U.S. Department of Commerce. NIST is one of the nation's oldest physical science laboratories.
NIST develops cybersecurity standards, guidelines, best practices, and other resources to meet the needs of U.S. industry, federal agencies and the broader public.
NIST also advances understanding and improves the management of privacy risks, some of which relate directly to cybersecurity.

SHA256 Checksum

Encryption is a technology used for data, at rest (stored) and in transit (transmitted), to reduce its vulnerability to tampering and other methods that could result in unexpected data alteration.
Linux distributions are often published and distributed together with "checksum" files, often signed to ensure that they have not been tampered
The checksum text file usually contain the checksums values of the ISO files in SHA256 format.
This common mechanism is used to prove that the files downloaded are what they claim to be.
This is particularly useful for ISOs downloaded from a third-party mirror or elsewhere the main publisher website.
A hash algorithm can be used to calculate digest messages of files, digests are used to detect if files have been changed since their digests were generated.
NIST states that the SHA-2 family of hash functions (SHA-224, SHA-256, SHA-384, SHA-512) may be used by Federal agencies for all applications using secure hash algorithms.
Source: https://csrc.nist.gov/Projects/Hash-Functions/NIST-Policy-on-Hash-Functions

About hash functions, collision and deprecated algorithms
MD5: The security of the MD5 hash function is severely compromised
Source: https://en.wikipedia.org/wiki/MD5#Collision_vulnerabilities
SHA-1: The algorithm has reached the end of its useful life, according to NIST that recommends professionals to migrate SHA-1 to SHA-2 or SHA-3 as soon as possible.
Source: https://www.nist.gov/news-events/news/2022/12/nist-retires-sha-1-cryptographic-algorithm

A Checksum is a cryptographic signature of a file that consists in a string (hash) that uniquely represents the file, so that if a file is tampered with, altered or modified, its checksum value will change letting the user know that the file has been changed.
There are many mathematical algorithms for generating file Checksums, SHA256 (Secure Hash Algorithm 256) is developed by the NSA (United States National Security Agency).
The algorithm breaks down the data of the file into small-sized parts and creates and combines the hash values for each part to create the checksum value.


Examples of commands to calculate files checksum (hash digest)

- MS Windows SHA256 checkum generation - CMD
C:\> certUtil -hashfile C:\file.iso SHA256

- Linux SHA256 checkum generation - Bash
sha256sum file.iso


Download ISO files (Distro example AlmaLinux 9.1)

Linux commands :

wget is a command that let us download files naming them as they are named on the server
The -O option can be used by wget to manually set the destination file name
eg. # wget URL # wget URL -O FILENAME

curl can also download files, it requires the -o option to set the name output (downloaded) file
eg. # curl URL -o FILENAME


Practical Example using Linux commands

1) Download an ISO file (e.g. Almalinux 9.1 minimal)
wget https://almalinux.mirror.garr.it/9/isos/x86_64/AlmaLinux-9.1-x86_64-minimal.iso

2) download of the CHECKSUM file
wget https://almalinux.mirror.garr.it/9/isos/x86_64/CHECKSUM

3) verify if the checksum of the downloaded file matches the one declared by the publisher in the CHECKSUM file
# The command "sha256sum AlmaLinux-9.1-x86_64-minimal.iso" calculates the checksum value of the file with SHA256 Algorithm,
because when a command is used within $() its output/value is encapsulated and usable by the next command in the pipeline (using pipe | ).

# The command grep is then used to search the checksum value within the CHECKSUM text file.
grep $(sha256sum AlmaLinux-9.1-x86_64-minimal.iso) CHECKSUM

The output of the command confirms that the value is found in the CHECKSUM file:
CHECKSUM:SHA256 (AlmaLinux-9.1-x86_64-minimal.iso) = 3eebd50aabe93b9cbea3937dcf5e1d893e397a36de128ae60a8d46ff048fa0d6

If there is no output, the calculated checksum value is not found in the CHECKSUM file, this can mean that the downloaded file is corrupted, incomplete or altered.
...or that there's something wrong in the command/parameters used.
