Linux and Bash resources

Manpages, help, suggestions, common terms, acronyms and more in Linux
The reader is invited to imagine that we can ...kind of... "listen" to Linux, as part of the open source and free software philosophy we embrace.
We simply have to read what Linux says in the shell, wise words of developers are spread as a precious source to understand systems.
Often the output of the commands itself contain details about errors, sintax explaination and suggestions.

Read. Everything.

Some programs and commands often provide options that are easily recognizable

Far from an universal rule, this intuition may help to create mental links about options, to help us keep in mind the ones we need often.
The following flags often mean something understandable in English language, but since there are differences between commands versions and shells, it is strongly suggested to check the command manpages and command options before executing commands. 
a - all
h - human readable
c - count
s - size, silent
o,O - output file
i - case insensitive

Options and chains - best practice

A good system administrator is able to find the best command with the best options to obtain the required result, with commands/chains as short as possible.

To master the Linux commmand line, we can not rely only on our memory, since programs and options may vary depending on the systems.

Luckily Linux comes with an extended and nice builtin documentation including the manual pages.


Common manual and help commands

May vary between distros, some could not available on all Linux systems

man - display a program's manual page (generally the preferred solution to read manual pages)
Man pages generally contain:
- title
- synopsis of the command's syntax;
- description of the command's purpose;
- listing and description of each of the command's options;
# man ls

whatis - display one-line manual page descriptions.
The whatis command displays the name and a one-line description of a man page matching the provided keyword.
# whatis ip

apropos - display appropriate commands.
The apropos command search the list of man pages for possible matches based on a search term.
# apropos ntp

info - display a program's info entry.
The GNU Project provides info as an alternative to man pages for their programs.
# info find

which - locates an executable file matching the keyword, searching in the user shell search path.
# which ps

type - find out whether the keyword it is built-in or external binary file.
# type ps

A great amount of commands support the --help option that is useful to display the syntax and common options of a command.
# ls --help # cd --help # free --help ...

To learn quickly and effectively how Linux works and to master the Bash commands, it's strongly suggested to consult the Linux manpages and to search information about commands.
With two basic commands we can understand the commands syntax, and discover what different combination of options can do.
man command
command --help
