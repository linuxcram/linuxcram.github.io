07_manpages

 Linux and Bash resources Manpages, help, suggestions, common terms, acronyms and more in Linux
We can think that there's a way to learn to "listen" to Linux, as part of the open source and free software philosophy we embrace.
We can do that readiing what Linux says, since these words are the voice of developers sharing with us tons of information that help to understand and use the systems.
Often the output of the commands contain information about errors, sintax and suggestions. We must read them.

Some programs and commands often provide options that are easily recognizable
This is not an universal rule, the aim is to provide a suggestion to create mental links to better memorize the options that better fit to reach our goals.
The following flags often mean something understandable in English language, but since there are differences between commands versions and shells, it is strongly suggested to check the command manpages and command options before executing commands.
a - all h - human readable c - count s - size, silent o,O - output file i - case insensitive

Options and chains - best practice
A good system administrator is able to find the best command with the best options to obtain the required result, using the shorter commands/chains possible.
To master the Linux commmand line, we can not only rely on our memory, also because programs and options may vary depending on the systems.
Luckily Linux comes with an extended and nice builtin documentation that help us reaching our goals.

Common manual and help commands Not all are available on every distro

man - display a program's manual page.
Man pages generally contain:
- title
- synopsis of the command's syntax
- description of the command's purpose
- listing and description of each of the command's options
eg. man ls

whatis - display one-line manual page descriptions The whatis command displays the name and a one-line description of a man page matching the provided keyword eg. whatis ip

apropos - display appropriate commands
The apropos command search the list of man pages for possible matches based on a search term.
eg. apropos ntp

info - display a program's info entry
The GNU Project provides info as an alternative to man pages for their programs.
eg. info find

which - locates an executable file matching the keyword, searching in the user shell search path.
eg. which ps


type - find out whether the keyword it is built-in or external binary file.
eg. type ps

A great amount of commands support the --help option that is useful to display the syntax and common options of a command.
eg. ls --help # cd --help # free --help ...

To learn in the best and effective way how Linux works and to master the Bash commands quickly
it is strongly suggested to search the information provided by Linux for the commands shown in these pages.
This will help the reader to understand, and hopefully memorize, the commands syntax and discover what different combination of options can do.
eg.
man command
command --help
